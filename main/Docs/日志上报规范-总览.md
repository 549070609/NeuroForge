## 概述

主动感知插件（Perception）接收结构化日志，对其中的事件进行分析，并根据事件级别触发预定义决策。感知器支持以下三种输入来源：

- **ATON / TOON 格式文本**：通过文件轮询或 Webhook 传入的结构化日志
- **Webhook JSON**：通过 HTTP POST 推送的 JSON payload
- **EventBus 事件**：通过内部事件总线订阅的日志事件

格式详细规范参见 [日志上报规范 · ATON 与 TOON 格式详解](./日志上报规范-ATON与TOON格式详解.md)。

## 采集方式

感知器支持四种独立的日志采集方式，可同时启用多种。

### 文件轮询

Cron 定时任务扫描指定目录下的日志文件，使用 `ReadLogsTool` 读取后送入解析流程。

```yaml
config:
  integration.perception:
    log_path: "./logs"
    cron_expr: "*/5 * * * *"
    filter_rules:
      level: ["error", "warn"]
```

| 配置项 | 类型 | 说明 |
|--------|------|------|
| `log_path` | `str` | 日志目录或文件路径 |
| `cron_expr` | `str` | 标准 Cron 表达式 |
| `filter_rules.level` | `list[str]` | 按级别预过滤，减少无效处理 |

### Webhook

上报方通过 HTTP POST 将日志推送至感知器端点，感知器实时解析并决策。

- **端点**：`POST /perception/webhook`
- **Content-Type**：`application/json`

```yaml
config:
  integration.perception:
    webhook:
      path: "/perception/webhook"
      secret: "YOUR_HMAC_SECRET"
```

### EventBus

感知器订阅内部事件总线，上报方发布 `log.written` 等事件时自动触发。payload 须包含 `log`（原始文本）或 `data`（已解析结构）字段。

```yaml
config:
  integration.perception:
    event_triggers:
      - "log.written"
      - "log.error"
```

### Agent 工具调用

Agent 在执行过程中可主动调用以下工具，无需额外配置：

| 工具名 | 说明 |
|--------|------|
| `read_logs` | 读取日志文件，支持目录 + 通配符 + 级别过滤 |
| `parse_log` | 将 ATON/TOON 文本解析为 Python 结构 |
| `perceive` | 对解析结果执行感知与决策 |
| `execute_decision` | 执行 `PerceptionResult` 中的决策 |

## 日志格式

感知器自动检测日志格式，无需上报方手动指定。检测优先级如下：

| 优先级 | 格式 | 识别规则 |
|--------|------|---------|
| 1 | **ATON** | 文本以 `@schema` 或 `@defaults` 开头 |
| 2 | **TOON** | 同时含有 `[`、`{`、`}:` 三个模式 |
| 3 | **JSON** | Webhook payload 中 `log` / `data` 字段为 dict |
| — | **未知** | 以上均不匹配，抛出 `ValueError` |

> **注意**：纯文本日志（非结构化）无法直接被感知器处理，须在上报前转换为 ATON 或 TOON 格式。

## 事件字段规范

每条日志事件（list 中的单个 dict）须满足以下字段约束，才能参与感知决策。

### 字段定义

| 字段名 | 必选 | 类型 | 说明 |
|--------|:----:|------|------|
| `level` | ✅ | `str` | 首选级别字段，取值见[日志级别](#日志级别与决策行为) |
| `message` | 推荐 | `str` | 事件描述文本，用于决策原因（`reason`）的展示 |
| `severity` | 可选 | `str` | `level` 的备用字段，语义相同，二选一 |
| `log_level` | 可选 | `str` | `level` 的备用字段，语义相同，二选一 |
| `type` | 可选 | `str` | `level` 的备用字段，语义相同，二选一 |
| `id`、`timestamp`、`source` 等 | 可选 | `any` | 扩展字段，透传至决策结果，不影响决策逻辑 |

感知器按 `level` → `severity` → `log_level` → `type` 顺序查找级别字段，取第一个非空值。

### 容器键名

顶层数据结构为 `dict` 时，感知器按以下优先级查找事件列表：

```
events  →  logs  →  records  →  data  →  items  →  首个 list 类型的值
```

顶层直接为 `list` 时，直接作为事件列表处理。

## 日志级别与决策行为

感知器按事件顺序遍历，命中第一条可触发级别后立即返回决策，不再处理后续事件。

| 级别 | 别名 | 默认决策 | 对应配置键 |
|------|------|----------|-----------|
| `error` | — | `find_user` | `error_triggers` |
| `warn` | `warning` | `find_user` | `warn_triggers` |
| `info` | — | 不触发（`none`） | — |
| `debug` | — | 不触发（`none`） | — |

**自定义触发行为**：通过 `rules` 参数覆盖默认决策目标：

```python
perceive(data, rules={
    "error_triggers": "execute",
    "warn_triggers": "call_agent",
})
```

可配置值：`find_user` · `execute` · `call_agent`

若需对 `info` / `debug` 级别触发决策，可通过 `levels` 扩展：

```python
perceive(data, rules={"levels": ["error", "warn", "info"]})
```

## 决策执行路径

感知决策产生后，由 `DecisionExecutor` 分发至对应执行路径。

### `find_user` — 通知用户

按以下顺序依次尝试，首个可用通道执行成功后停止：

1. **EventBus**：向 `perception.alert` 事件发布通知 payload
2. **notify_callback**：调用注册的回调函数，传入 `(PerceptionResult, content_str)`
3. **日志记录**：无通道可用时写入 warning 日志

### `execute` — 自动执行

根据 `execute_actions` 配置执行预定义动作：

```yaml
execute_actions:
  default:
    type: shell
    cmd: "bash /opt/scripts/alert.sh"
  # 或
  default:
    type: http
    url: "https://example.com/alert"
    method: POST
    headers:
      Authorization: "Bearer TOKEN"
    body:
      event: "perception_alert"
```

| `type` | 必填参数 | 说明 |
|--------|----------|------|
| `shell` | `cmd` | 字符串或参数列表，异步执行 |
| `http` | `url` / `endpoint` | 支持任意 HTTP 方法，自动编码 JSON body |

### `call_agent` — 委派 Agent

通过 `engine.run(prompt)` 将告警信息委派给目标 Agent 处理：

```yaml
call_agent_config:
  target_agent: "ops-agent"
  prompt_template: |
    Handle this perception alert.

    Reason: {reason}
    Data: {data}
```

`prompt_template` 支持 `{reason}`、`{data}`、`{metadata}` 三个占位符。若未配置，使用内置默认模板。

### `none` — 无操作

无可触发事件时返回，不执行任何动作。

## 错误处理

| 情况 | 行为 |
|------|------|
| 事件缺少所有级别字段（`level` / `severity` / `log_level` / `type`） | 该事件被跳过，继续处理后续事件 |
| 日志格式无法识别 | `parse_log()` 抛出 `ValueError`，需上报方检查格式 |
| 全部事件均为 `info` / `debug` | 决策为 `none`，`DecisionExecutor` 直接返回成功 |
| Webhook payload 缺少 `log` / `data` / `raw` 字段 | 返回 `{"status": "skipped", "reason": "no log data in payload"}` |
| `execute` 模式下无匹配 `execute_actions` 配置 | 返回执行失败，`message: "No execute action configured"` |
| `call_agent` 模式下无 engine 及 engine_factory | 返回执行失败，`message: "No engine or engine_factory for call_agent"` |
