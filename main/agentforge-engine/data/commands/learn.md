# 学习笔记提取

从当前会话中提取非显而易见的学习内容，写入项目中的 AGENTS.md 文件，构建代码库知识库。

## 用法

```
/learn                - 分析当前会话，提取学习内容
/learn --scope src/   - 限定在特定目录
```

## 参数

$ARGUMENTS

---

## 学习内容类型（应该记录的）

### ✅ 应该记录

- **隐藏的文件/模块关系**: 不明显的依赖关系
- **非标准的执行路径**: 特殊的调用链或工作流
- **非标准配置**: 环境变量、配置文件的隐藏要求
- **误导性错误消息**: 实际原因与错误信息不符的情况
- **API/工具的特性**: 未文档化的行为或变通方案
- **未文档化的构建命令**: README 中没有的构建/测试命令
- **架构决策**: 为什么这样设计，有什么约束
- **必须一起更改的文件**: 耦合的文件组合

### ❌ 不应该记录

- 文档中明确说明的事实
- 标准的语言/框架行为
- 已在 AGENTS.md 中的内容
- 冗长的解释（保持 1-3 行）
- 会话特定的临时细节

## 执行步骤

### 1. 分析会话历史

回顾当前会话中的交互，识别：
- "啊哈"时刻（突然理解的事情）
- 花费时间调试的问题
- 需要多次尝试才成功的操作
- 非显而易见的解决方案

### 2. 确定适用范围

根据学习内容的影响范围，确定存储位置：
- **项目级**: 影响整个项目 → `AGENTS.md`
- **模块级**: 影响特定模块 → `<模块>/AGENTS.md`
- **功能级**: 影响特定功能 → `<功能目录>/AGENTS.md`

### 3. 读取现有 AGENTS.md

如果文件已存在，读取内容避免重复。

### 4. 创建或更新文件

按照以下格式添加学习内容：

```markdown
# AGENTS.md - 项目知识库

## 架构

- [日期] 模块 A 和模块 B 共享配置，修改时需同时更新
- [日期] 事件系统使用发布-订阅模式，注意内存泄漏风险

## 构建和测试

- [日期] 运行完整测试需要先设置 TEST_ENV=1
- [日期] 某些测试需要网络访问，离线环境会失败

## 已知问题

- [日期] 错误消息 "X not found" 实际上是配置问题，检查 .env 文件
- [日期] Windows 上路径分隔符问题，使用 os.path.join 而非硬编码

## API 和工具

- [日期] 模型 X 的 temperature 参数实际范围是 0.0-2.0，文档错误
- [日期] 某个 API 在高并发下会限流，建议添加重试逻辑
```

## 知识分层示例

```
项目根目录/AGENTS.md
  → 项目级别的学习内容
  → 跨模块的依赖关系
  → 构建和部署注意事项

packages/auth/AGENTS.md
  → auth 模块的学习内容
  → 认证相关的配置
  → 安全注意事项

src/api/endpoints/AGENTS.md
  → API endpoints 的学习内容
  → 端点之间的依赖
  → 版本兼容性
```

---

## 写作原则

1. **简洁**: 每条记录 1-3 行
2. **可操作**: 提供具体的行为指导
3. **可发现**: 使用清晰的分类和标题
4. **避免冗余**: 不重复文档中已有的内容

---

## 示例

### 好的学习记录

```markdown
## 依赖关系

- [2026-02-21] config.py 和 database.py 必须同时修改，否则会导致连接失败
- [2026-02-21] 修改 API 路由时，需要同步更新 openapi.yaml
```

### 不好的学习记录

```markdown
## 依赖关系

- Python 是一种编程语言（这是常识，不需要记录）
- 这个项目使用 pytest 进行测试（这在 README 中已说明）
- 详细的五页解释为什么这样设计（太长，应该精简）
```
