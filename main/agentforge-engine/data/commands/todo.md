# 任务管理

管理待办事项和任务列表。

## 使用方式

- `/todo` - 显示当前任务列表
- `/todo add [任务描述]` - 添加新任务
- `/todo done [任务ID]` - 标记任务完成
- `/todo clear` - 清空任务列表

## 任务操作

### 1. **查看任务**

显示任务列表：
```markdown
# TODO List

## 进行中
- [ ] 1. 实现用户认证功能 (优先级: 高)
- [ ] 2. 优化数据库查询 (优先级: 中)

## 待办
- [ ] 3. 添加单元测试
- [ ] 4. 更新文档

## 已完成
- [x] 5. 初始化项目结构 ✓ (2026-02-21)
```

### 2. **添加任务**

创建新任务：
- 分配唯一 ID
- 记录创建时间
- 设置优先级（高/中/低）
- 添加标签（可选）

### 3. **更新任务状态**

- 标记为进行中
- 标记为已完成
- 更新任务描述
- 调整优先级

### 4. **任务统计**

显示：
- 总任务数
- 已完成数
- 进行中数
- 待办数
- 完成率

---

## 任务格式

每个任务包含：
```markdown
- [ ] ID. 任务描述
  - 优先级: 高/中/低
  - 标签: #功能 #优化
  - 创建: 2026-02-21
  - 备注: 额外说明
```

---

## 最佳实践

1. **任务粒度**
   - 任务应具体、可执行
   - 避免过大或模糊的任务

2. **优先级设置**
   - 高: 阻塞性问题、紧急需求
   - 中: 常规功能、改进
   - 低: 优化、重构

3. **定期回顾**
   - 每日查看任务列表
   - 及时更新状态
   - 清理过期任务
