## 🎯 核心概念

### Agent 本质

```
Agent = LLM + Context + Tools + Loop
```

**四个要素：**
1. **LLM** - 大语言模型（决策大脑）
2. **Context** - 上下文管理（记忆）
3. **Tools** - 工具集（能力）
4. **Loop** - 执行循环（行为）

### Agent 类型

| 类型 | 特征 | 适用场景 |
|------|------|---------|
| **单一职责** | 单角色，独立完成 | 简单任务 |
| **协调者** | 管理子Agent | 复杂多步骤 |
| **层级** | 树状结构 | 大型项目 |

### 构建流程

```
定义 → 实现 → 测试 → 优化
  ↓      ↓      ↓      ↓
 4步    3步    2步    可选
```

## 📋 核心模式速查

### System Prompt 结构

```markdown
# [角色]

## 角色定义
## 核心职责
## 工作流程
## 输出格式
## 工作原则
## 限制
```

### ReAct Loop 核心

```python
while iteration < MAX:
    response = llm_call(messages, tools)

    if not response.has_tool_calls:
        return response.text

    for tool_call in response.tool_calls:
        result = execute_tool(tool_call)
        context.add_tool_result(tool_call.id, result)
```

### 子Agent调用

```python
# 1. 深度检查
if depth >= max_depth:
    return "Error"

# 2. 创建隔离上下文
sub_context = ContextManager(system_prompt=config["prompt"])

# 3. 过滤工具
sub_tools = filter_tools(allowed=config["tools"])

# 4. 递归注册Task
if depth + 1 < max_depth:
    sub_tools.register(TaskTool(depth+1, max_depth))

# 5. 执行
result = await sub_engine.run(prompt)
```

### 输出格式

```xml
</错误>
```

## 🎓 核心原则

1. **单一职责** - 每个Agent只做一件事
2. **最小权限** - 只给必需工具
3. **上下文隔离** - 子Agent独立上下文
4. **深度控制** - 防止无限递归
5. **结果回传** - 工具结果必须回传
6. **错误处理** - 优雅处理异常
7. **成本控制** - 统计Token消耗

## 📊 关键模式对比

### 上下文管理

| 方式 | 优点 | 缺点 | 适用 |
|------|------|------|------|
| **截断** | 简单快速 | 丢失信息 | 对话历史 |
| **压缩** | 保留要点 | 需要LLM | 长文本 |
| **摘要** | 精炼 | 额外成本 | 重要信息 |

### 子Agent调用

| 方式 | 说明 | 适用 |
|------|------|------|
| **顺序** | 一个接一个 | 有依赖关系 |
| **并行** | 同时执行 | 独立任务 |
| **条件** | 根据结果决定 | 动态流程 |

### 错误处理

| 层级 | 处理方式 |
|------|---------|
| **工具** | try-catch，返回错误XML |
| **循环** | 检查错误结果，继续或中止 |
| **系统** | 全局异常处理，优雅降级 |
